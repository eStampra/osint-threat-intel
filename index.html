// 5 GITHUB-READY DELIVERABLES — BUILT STEP BY STEP
◈ STRATEGY: Each project goes on GitHub as a polished markdown write-up. Recruiters check your GitHub. These 5 projects demonstrate: OSINT methodology, threat actor analysis, phishing/IOC analysis, geolocation investigation, and full CTI reporting — exactly what analysts do on Day 1.
PROJECT 1 · PHASE 1 · WEEKS 3–4
OSINT Footprinting Report — Public Target
▾
Objective: Perform a complete passive OSINT footprint on a public company. Use a small real business (not a major corporation). Document every finding methodically to analyst standard.
Tools: theHarvester, Shodan (free), WHOIS / who.is, DNSdumpster, Google Dorking, LinkedIn, Wayback Machine, Hunter.io (free tier)
1
Choose target: small business, non-sensitive sector. Write a rules-of-engagement.md file — scope, what you will/won't do, disclaimers. This shows professionalism to recruiters.
2
Run theHarvester -d [domain] -b all in Kali. Log all emails, subdomains, and IPs. Screenshot output.
3
Search Shodan.io for the company's IP range. Document exposed services, open ports, SSL certificate data.
4
WHOIS via who.is — note registrar, creation date, name servers. Try reverse WHOIS to find associated domains.
5
Google Dorking: site:[domain], filetype:pdf site:[domain], "@[domain]" filetype:xlsx. Document any sensitive exposures found.
6
LinkedIn search for employees. Note job titles and any tech stack mentions in profiles (reveals internal tools).
7
Wayback Machine on the domain — look for old exposed data or removed sensitive pages.
8
Write report: Executive Summary → Methodology → Findings + Screenshots → Risk Assessment → Recommendations. Push to GitHub.

PROJECT 2 · PHASE 2 · WEEKS 6–7
APT Threat Actor Profile — MITRE ATT&CK Mapped
▾
Objective: Research a real documented APT group and produce an analyst-grade threat actor profile with ATT&CK navigator map. Choose one: Lazarus Group, APT28, Cozy Bear, or Sandworm.
Tools: MITRE ATT&CK Groups, ATT&CK Navigator, Malpedia, AlienVault OTX, CISA advisories, open-source reporting
1
Read your chosen group's full MITRE ATT&CK page. List every technique under each tactic.
2
Open ATT&CK Navigator. Create a layer highlighting your group's techniques. Export as JSON and SVG for GitHub.
3
Research 3 documented campaigns by this group. For each: date, target sector, initial access method, tools used, published IOCs.
4
Search AlienVault OTX for related pulses. Note any live IOC feeds.
5
Write the profile: Attribution → Motivation → Campaigns → TTP Table → IOC Sample → Detection Recommendations.
6
Add a Defensive Recommendations section — shows you understand blue-team application of CTI.

PROJECT 3 · PHASE 2–3 · WEEKS 9–10
Phishing Email Analysis — End-to-End IOC Extraction
▾
Objective: Analyse a real phishing email, extract all IOCs, sandbox the payload, and write a structured analyst report. Source samples from PhishTank or public repos.
Tools: PhishTank, Any.run sandbox, VirusTotal, URLscan.io, MXToolbox header analyser
1
Source a phishing email from PhishTank or phishing_pot on GitHub. Never open in a live environment.
2
Extract and analyse email headers via MXToolbox. Note originating IP, mail path, SPF/DKIM/DMARC pass or fail.
3
Extract all URLs. Submit each to URLscan.io and VirusTotal. Document scores, redirect chains, hosting provider.
4
Use Any.run public submissions for a known malware sample. Screenshot: network connections, file drops, registry changes.
5
Compile IOC table: IPs, domains, URLs, file hashes (MD5/SHA256), sender email addresses.
6
Write report: Incident Summary → Header Analysis → Payload Analysis → IOC Table → Mitigation Steps.

PROJECT 4 · PHASE 3 · WEEKS 12–13
Geolocation OSINT Investigation — Bellingcat Methodology
▾
Objective: Take a publicly available image from a known challenge and geolocate it using only open-source methods. Document your full reasoning chain.
Tools: Google Maps/Street View, SunCalc, Yandex Reverse Image Search, ExifTool, Wikimapia, geolocation.ninja
1
Source a challenge image from Sofia Santos OSINT exercises — use a geolocation challenge.
2
Run ExifTool: exiftool image.jpg. Check for GPS data, camera model, timestamps. Note what's present or has been stripped.
3
Identify visual clues: language on signs, architectural style, vegetation, road markings, vehicle types, sky colour and sun angle.
4
Run reverse image searches on Google and Yandex (Yandex is superior for Eastern European / non-Western content).
5
If shadows are visible, use SunCalc to narrow down location and time of day from shadow angle.
6
Confirm via Google Street View and satellite imagery. Document confidence level at each step.
7
Write up: Objective → Evidence Chain (numbered, with screenshots) → Conclusion → Confidence Level Assessment.

PROJECT 5 · PHASE 3–4 · WEEKS 14–15
Full Threat Intelligence Report — Current Threat
▾
Objective: Produce a full strategic and tactical threat intelligence report on a current threat — ransomware campaign, sector-specific threat, or recent major incident. This is your flagship portfolio piece.
Tools: All prior tools + CISA advisories, NCSC reports, Mandiant/Secureworks public blogs, ATT&CK Navigator
1
Choose a recent incident (within 12 months). Sources: CISA Alerts, NCSC Threat Reports.
2
Write an Executive Summary — 1 page max, suitable for a non-technical manager. This is the most important skill CTI analysts are hired for.
3
Technical Analysis: TTPs mapped to ATT&CK, IOC table, infrastructure analysis using Shodan/Censys lookups.
4
Attribution section: known or suspected actor(s) with confidence levels — High/Medium/Low. Use intelligence assessment language.
5
Impact Assessment: affected sectors, geographic scope, estimated business impact.
6
Mitigation and Recommendations: CISA-aligned mitigations, detection guidance, defensive posture advice.
7
Polish to professional standard. Get feedback on r/cybersecurity or a Discord community before publishing.
